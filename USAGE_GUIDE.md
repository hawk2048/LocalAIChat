## 快速开始

### 第一步：准备模型

LocalAI Chat 需要 GGUF 格式的模型文件。推荐模型：

| 设备配置 | 推荐模型 | 大小 | 说明 |
|----------|----------|------|------|
| 入门级 (4GB RAM) | Qwen2.5-1.5B-Instruct-Q4_K_M | ~1GB | 适合快速测试 |
| 标准级 (6GB RAM) | Qwen2.5-3B-Instruct-Q4_K_M | ~2GB | 性能与速度平衡 |
| 高端机 (8GB+ RAM) | Qwen2.5-7B-Instruct-Q4_K_M | ~4GB | 最佳效果 |

**模型下载来源：**
- [Hugging Face - Qwen2.5](https://huggingface.co/Qwen/Qwen2.5-7B-Instruct-GGUF)
- [Hugging Face - Llama 3.2](https://huggingface.co/bartowski/Llama-3.2-3B-Instruct-GGUF)

### 第二步：放置模型

将下载的 `.gguf` 文件放入设备存储：

```
/sdcard/Download/models/
├── Qwen2.5-7B-Instruct-Q4_K_M.gguf
├── Llama-3.2-3B-Instruct-Q4_K_M.gguf
└── ...
```

### 第三步：加载模型

1. 打开应用，点击右上角 **设置图标** ⚙️
2. 点击 **"选择模型"**
3. 浏览并选择 `.gguf` 文件
4. 等待模型加载完成（首次加载可能需要 10-30 秒）

### 第四步：开始聊天

在输入框输入问题，点击发送即可开始对话！

## 模型管理

### 模型选择界面

进入 **设置 → 模型管理** 可以：

- 📂 **选择模型** - 从设备存储选择 GGUF 文件
- ℹ️ **查看模型信息** - 显示模型详细参数
- 🗑️ **卸载模型** - 释放内存（不删除文件）

### 模型信息说明

加载模型后，可以查看以下信息：

| 属性 | 说明 |
|------|------|
| 模型名称 | 文件名 |
| 参数规模 | 如 7B、3B、1.5B |
| 量化类型 | Q4_K_M、Q5_K_M 等 |
| 上下文长度 | 最大支持的 token 数 |
| 词表大小 | Vocabulary 大小 |
| 层数/头数 | 模型架构参数 |

### 模型切换

应用支持运行时切换模型：

1. 进入设置 → 模型管理
2. 选择新模型
3. 系统会自动释放旧模型，加载新模型

> ⚠️ **注意**: 切换模型会清除当前对话历史

## 聊天功能

### 基本聊天

1. **输入问题** - 在底部输入框输入文本
2. **发送消息** - 点击发送按钮或按回车键
3. **等待回复** - AI 会流式生成回复
4. **继续对话** - 可以追问，系统会记住上下文

### 流式生成

LocalAI Chat 支持**实时流式输出**：

- 文字逐字显示，无需等待完整回复
- 可以随时点击 **"停止"** 按钮中断生成
- 适合长回复场景

### 对话历史

- 对话自动保存，关闭应用不会丢失
- 支持多轮对话，上下文保持
- 可以在设置中清除历史记录

### 清除对话

点击右上角菜单 → **"清除对话"**

> 这会清除当前会话的所有消息，但不会清除模型上下文

## 参数配置

进入 **设置 → 生成参数** 调整 AI 行为。

### 核心参数

#### Temperature (温度)

```
范围: 0.1 - 2.0
默认: 0.7
```

| 值 | 效果 | 适用场景 |
|----|------|----------|
| 0.1 - 0.3 | 保守、稳定 | 代码生成、事实问答 |
| 0.4 - 0.7 | 平衡 | 日常对话 |
| 0.8 - 1.0 | 创意、随机 | 创意写作、头脑风暴 |
| 1.0+ | 高度随机 | 实验性用途 |

#### Top-P (核采样)

```
范围: 0.1 - 1.0
默认: 0.9
```

控制候选词的概率累积阈值：
- **低值 (0.5)**: 更精确，但可能重复
- **高值 (0.9)**: 更多样，但可能偏离主题

#### Top-K

```
范围: 1 - 100
默认: 40
```

限制每步考虑的候选词数量：
- **低值 (10)**: 更确定
- **高值 (100)**: 更多样

#### Repeat Penalty (重复惩罚)

```
范围: 1.0 - 2.0
默认: 1.1
```

防止重复生成：
- **1.0**: 无惩罚
- **1.1 - 1.2**: 适度惩罚（推荐）
- **1.5+**: 强惩罚，可能导致语法问题

### 高级参数

#### Max Tokens (最大生成长度)

```
范围: 64 - 4096
默认: 2048
```

控制单次回复的最大 token 数。建议：
- 短回复: 256 - 512
- 正常对话: 1024 - 2048
- 长文生成: 4096

#### Context Length (上下文长度)

```
范围: 512 - 8192
默认: 4096
```

模型能记住的对话历史长度：
- **影响内存占用**: 越大越占内存
- **影响对话连贯性**: 越大越连贯

#### Threads (推理线程)

```
范围: 1 - CPU核心数
默认: 4
```

CPU 推理线程数：
- 一般设置为 CPU 大核心数
- 过多线程可能反而降低性能

## 高级功能

### 自定义 System Prompt

在 **设置 → System Prompt** 中设置 AI 的行为：

```
默认: 你是一个有帮助的AI助手。

示例（代码助手）:
你是一个专业的编程助手。回答问题时：
1. 提供清晰的代码示例
2. 解释关键概念
3. 指出潜在问题

示例（翻译助手）:
你是一个专业翻译。请将用户输入翻译成英文，保持原文风格和语调。
```

### 模型预加载

如果希望应用启动时自动加载模型：

1. 在设置中勾选 **"自动加载上次模型"**
2. 下次启动会自动加载上一次使用的模型

### 内存优化

对于内存较小的设备：

1. **减少上下文长度** - 设置为 2048 或更低
2. **使用小模型** - 1.5B 或 3B 参数
3. **启用内存映射** - 设置中开启 mmap

## 故障排除

### 常见问题

#### 1. 模型加载失败

**症状**: 显示 "模型加载失败" 或闪退

**解决方案**:
- 确认文件是有效的 GGUF 格式
- 检查文件是否完整（大小是否正确）
- 尝试重新下载模型
- 查看日志: `adb logcat | grep LocalAI`

#### 2. 生成速度慢

**解决方案**:
- 增加推理线程数
- 使用更小的量化模型 (Q4_K_M → Q3_K_M)
- 减少上下文长度
- 关闭其他应用释放内存

#### 3. 回复质量差

**解决方案**:
- 调整 Temperature (通常 0.7 效果好)
- 尝试不同的模型
- 使用更大的模型 (7B vs 3B)
- 使用更高质量的量化 (Q5_K_M vs Q4_K_M)

#### 4. 内存不足 (OOM)

**解决方案**:
- 使用参数更小的模型
- 减少上下文长度
- 关闭其他应用
- 使用量化更高的模型

#### 5. 回复中断

**症状**: 生成中途停止

**解决方案**:
- 检查 Max Tokens 设置
- 可能触发了 EOS (结束标记)
- 尝试调整 Repeat Penalty

| Snapdragon 8 Gen 1 | Qwen2.5-3B-Q4 | 4096 | ~15-20 |
